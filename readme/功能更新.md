# 功能更新日志

## 2026-06-22 - 修复工具栏图标偶发加载失败问题

### 修复
- 修复 `ToolbarButton.cs` 中图标在场景切换后偶发加载失败的问题
  - 根因：场景切换后新的 `ToolbarButton` 实例立即尝试加载图标，但此时 `GameDatabase` 可能尚未完全就绪，`GetTexture` 返回 null。
  - 修复方式：
    - 在 `LoadIconTexture()` 中增加 `GameDatabase.Instance == null` 检查
    - 图标加载失败时使用 `Debug.LogWarning` 替代 `Debug.LogError`，并说明会在下一帧重试
    - `SetupAppLauncher()` 保持失败即返回、下帧重试的逻辑
    - `Cleanup()` 中不再 `Destroy(iconTexture)`，避免破坏 GameDatabase 内部引用
  - 涉及文件：`ToolbarButton.cs`

## 2026-06-22 - 实现音量控制、多语言i18n和发射前安全检查

### 新增
- 音量控制功能（无预览）
  - 新增 `SettingsManager.cs` 设置管理器
    - 使用 ConfigNode 将音量设置保存到当前存档目录 `saves/<存档名>/KSPLaunchCountdown/Settings.cfg`
    - 默认音量 100%，支持 0%~100% 范围
  - 修改 `AudioPlayer.cs`
    - 新增 `Volume` 属性，实时控制 AudioSource 音量
  - 修改 `CountdownMenu.cs`
    - 在菜单中添加音量滑块，拖动时实时保存并应用音量
  - 修改 `KSPLaunchCountdownMod.cs`
    - 初始化 SettingsManager 并将音量应用到 AudioPlayer
- 多语言 i18n 支持（中文/英文/俄文）
  - 新增 `Localization.cs` 本地化管理器
    - 自动检测 KSP 当前语言（使用 `using KSP.Localization;` 直接访问 `Localizer.CurrentLanguage`）
    - 从 `GameData/KSPLaunchCountdown/Localization/*.cfg` 加载翻译
    - 支持回退到英文，未找到翻译时返回键名
  - 新增语言文件
    - `GameData/KSPLaunchCountdown/Localization/zh-cn.cfg` 简体中文
    - `GameData/KSPLaunchCountdown/Localization/en-us.cfg` 英文
    - `GameData/KSPLaunchCountdown/Localization/ru-ru.cfg` 俄文
  - 修改 `CountdownMenu.cs`
    - 所有界面文本替换为本地化键
  - 修改 `KSPLaunchCountdownMod.cs` / `CountdownController.cs`
    - 注入 Localization 实例
- 发射前安全检查（v0.2.3 规划内容）
  - 新增 `LaunchSafetyChecker.cs` 安全检查器
    - 检查是否在发射台（高度 < 100m 且速度接近 0）
    - 检查是否已有正在进行的倒计时
    - 检查飞船电量：当前电量小于总电量的 1/20 时拒绝发射
    - 检查芯一级发动机是否已启动
    - `SafetyCheckResult` 记录 `LowElectricCharge` 标志，用于SAS失败时区分MJ控制和停电
  - 修改 `CountdownController.cs`
    - 根据安全检查结果决定是否采用"先点火后放行"策略
    - 发动机已启动时：倒计时期间保持 0 油门，音频结束后只加满油门，不执行自动分级
    - 发射前准备序列中尝试开启SAS最多3次：
      - 3次后仍失败且电量正常：判断为MJ控制，继续正常执行分级
      - 3次后仍失败且电量低：判断为停电，中止发射序列
    - 新增 `OnSafetyCheckFailed` 事件，确保内部安全检查失败时也能通知UI显示警告
  - 修改 `LaunchSequence.cs`
    - 新增 `SetThrottle(float)` 通用方法
    - 新增 `SetZeroThrottle()` 快捷方法
    - 修改油门保持机制，支持任意目标油门值
  - 修改 `CountdownMenu.cs`
    - 点击 Launch 时执行安全检查
    - 检查未通过时在菜单内显示警告和"强制发射"复选框
    - 勾选强制发射后可跳过检查启动倒计时
    - 订阅 `OnSafetyCheckFailed` 事件，确保任何路径的检查失败都会显示警告

## 2026-06-22 - 修复工具栏按钮不显示问题

### 修复
- 修复 `ToolbarButton.cs` 工具栏按钮在飞行场景不显示的问题
  - 根因：本地 stub 类型 `KSPLaunchCountdown.ApplicationLauncher` 遮蔽了真实的 `KSP.UI.Screens.ApplicationLauncher`，导致运行时始终调用 stub 的默认值（`Ready=false`、`Instance=null`），无法真正注册按钮。
  - 修复方式：删除本地 stub，改为直接调用 `KSP.UI.Screens.ApplicationLauncher`。
  - 参考示例：`StagingAndEngineConfig.cs.example` 中的 `ApplicationLauncher.Instance != null` 检查、`AddModApplication` 注册、`RemoveModApplication` 清理模式。
- 更新 `KSPLaunchCountdown.csproj`
  - 添加 `UnityEngine.AnimationModule.dll` 引用，解决 `ApplicationLauncherButton` 的间接编译依赖。
- 更新 `KSPApiHelper.cs` 头部注释
  - 说明 `ApplicationLauncher` / `ApplicationLauncherButton` 已改为直接调用，不再使用 stub 或反射。
- 更新 `README.md`
  - 修正“工具栏按钮注册”设计决策的说明，与当前直接调用实现保持一致。
